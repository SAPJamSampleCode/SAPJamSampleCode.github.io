Manipulating GET responses

Open-ended GET requests (GET requests that do not specify a particular resource) return a feed (a series) of entries (resources). The SAP Jam OData API will return a maximum of 20 entries in a feed, but there are mechanisms for retrieving results beyond the first 20 entries, for sorting them according to various property values, or for filtering them to show only the entries that you are interested in. You can also limit or expand the amount of information that is returned. This page explains how to manipulate the returned results using a variety of system query options.

Manipulating GET response information from the GET request is accomplished in OData using "system query options" that are appended to the API call URL following a question mark (?), and which are each preceded with a dollar sign ($). This page will explain how to:

  • Use $count to discover how many entries there are
  • Use $top to set the number of entries returned
  • Use $skip to offset the set of entries returned
  • Use $skiptoken to offset the set of entries returned
  • Use $filter to limit the entries returned according to their content
  • Use $orderby to set the criteria that the returned entries are ordered by
  • Use $select to limit and select the data that is shown for the returned entries
  • Use $expand to include details on any property for which there are available navigations

Use $count to discover how many entries there are

To find the count of the total number of entries that exist for a particular resource type or entity, use the $count system query option, as shown in the following curl command.

Note: While most of the operations described in this page are "system query options" that are appended to the API call URL following a question mark (?), $count is considered to be a "resource path segment", so it is appended to the API call URL with a slash (/).
curl -i -H 'Accept: application/xml' -H 'Content-Type: application/xml'
     -H "Authorization: Bearer <!--[An OAuth2.0 auth_token has been removed from here.]-->"
     "https://<jam#>.sapjam.com/api/v1/OData/Groups/$count"

This will return only a number (in addition to the curl and HTTP responses, such as the HTTP status code, which will be "200 OK" for a successful response), as is shown below:

Use $top to set the number of entries returned

By default, the SAP Jam OData API returns a maximum of 20 resource entries in response to a collection-valued request. However, you can use $top to set the number of returned entries to any value between 1 and 100. For example, you many want to show a maximum of six results per page in a UI widget, so you might use a call similar to the cur command shown below:

curl -i -H 'Accept: application/xml' -H 'Content-Type: application/xml'
     -H "Authorization: Bearer <!--[An OAuth2.0 auth_token has been removed from here.]-->"
     "https://<jam#>.sapjam.com/api/v1/OData/Groups?$top=6"

This call would return only the first six results, which you could then page through for subsequent sets of six results.

       xmlns:m="http://schemas.microsoft.com/ado/2007/08/dataservices/metadata">

    <link rel="self" title="Groups"

SAP Jam Limitation: $top is not available for FeedEntries or Notifications API calls.

Use $skip to offset the set of entries returned

Use the $skip system query option to perform paging of results. This is done by setting the number of results to "skip" in each subsequent command. To illustrated this, $skip is combined with the $top scenario described in the previous section. This usage is shown in the following curl command:

curl -i -H 'Accept: application/xml' -H 'Content-Type: application/xml'
     -H "Authorization: Bearer <!--[An OAuth2.0 auth_token has been removed from here.]-->"
     "https://<jam#>.sapjam.com/api/v1/OData/Groups?$top=6&$skip=6"

This request will show the next six results, from group 7 to group 12, inclusive. Requests for subsequent pages of six results would increase the value of skip accordingly (12, 18, 24, and so on).

No code example of the returned results is shown, as it would appear very much like the example in the $top section above.

SAP Jam Limitation: $skip is not available for FeedEntries or Notifications API calls.

Use $skiptoken to offset the set of entries returned beginning from the specified entry

As has been stated, the SAP Jam API returns a maximum of the 20 resource entries to any collection-valued API GET request, but when an SAP Jam OData GET response does return that maximum of 20 entries from a larger collection of resources, the response shows a URL with a $skiptoken for calling the next 20 results, which can be seen in lines 9-11 in the following response example:

<?xml version="1.0" encoding="UTF-8"?>
<feed xml:base="https://<jam#>.sapjam.com/api/v1/OData" xmlns="http://www.w3.org/2005/Atom"
      xmlns:d="http://schemas.microsoft.com/ado/2007/08/dataservices"
      xmlns:m="http://schemas.microsoft.com/ado/2007/08/dataservices/metadata">
    <title>Groups</title>
    <id>https://<jam#>.sapjam.com/api/v1/OData/Groups</id>
    <link rel="self" title="Groups" href="https://<jam#>.sapjam.com/api/v1/OData/Groups"/>
    <link rel="next"
      title="Groups?%24skiptoken=6jh0pc7h5052xa7e5g9mzjtwcoqvz0n46g1vpp073we8gx9nokqozzzgc8i4dmdp9hheopawa"
      href="Groups?%24skiptoken=6jh0pc7h5052xa7e5g9mzjtwcoqvz0n46g1vpp073we8gx9nokqozzzgc8i4dmdp9hheopawa"/>
    <updated>2015-04-22T20:06:10+00:00</updated>
    <entry>
        <!--[Entry details have been removed from here.]-->
    </entry>
    <!--[Multiple Group entries would appear here.]-->
</feed>

To call that URL, you would have to append it to the "service root URI", as is a shown in the following example curl command:

curl -i -H 'Accept: application/xml' -H 'Content-Type: application/xml'
     -H "Authorization: Bearer <!--[An OAuth2.0 auth_token has been removed from here.]-->"
     "https://<jam#>.sapjam.com/api/v1/OData/
         Groups?$skiptoken=6jh0pc7h5052xa7e5g9mzjtwcoqvz0n46g1vpp073we8gx9nokqozzzgc8i4dmdp9hheopawa"

The above request would return the next 20 results of the initially requested resource type.

SAP Jam Limitation: $skiptoken is server-side paging, which cannot be used with the other options in this page that are client-driven requests.

Use $filter to limit the entries returned according to their content

The $filter system query option allows you to limit the returned entries to only those whose properties match the values that you want to see, as is illustrated in the following curl command.

curl -i -H 'Accept: application/xml' -H 'Content-Type: application/xml'
     -H "Authorization: Bearer <!--[An OAuth2.0 auth_token has been removed from here.]-->"
     "https://<jam#>.sapjam.com/api/v1/OData/Groups?$filter=Name eq 'Customer Support Group'"

Use $orderby to set the criteria that the returned entries are ordered by

The $orderby system query option allows you to set the criteria by which your results are sorted, although there are limitations in the SAP Jam API on which entities and properties can be used to set the sort order. An example of the $orderby usage is shown in the following curl command:

curl -i -H 'Accept: application/xml' -H 'Content-Type: application/xml'
     -H "Authorization: Bearer <!--[An OAuth2.0 auth_token has been removed from here.]-->"
     "https://<jam#>.sapjam.com/api/v1/OData/
         Discussions('0MlgLVpePpCntnEWUU1kW2')/Comments?$orderby=LastModifiedAt asc"

Note the use of "asc" in the $orderby request above; this indicates that the results should be returned in "ascending" order. Alternatively, you could use "desc" to have the results returned in "descending" order. The above request would return results similar to the following example response.

Use $select to limit the data that is shown for the returned entries

Use the $select option if you want to limit the information that is returned from a GET request. For example, if you are only interested in retrieving the IDs of Groups, you can run the API call shown in the following curl command:

curl -i -H 'Accept: application/xml' -H 'Content-Type: application/xml'
     -H "Authorization: Bearer <!--[An OAuth2.0 auth_token has been removed from here.]-->"
     "https://<jam#>.sapjam.com/api/v1/OData/Groups?$select=Id"

The returned result would look something like the following:

Use $expand to include details on any property for which there are available navigations

The $expand system query option, as illustrated in the following curl command, allows you to get the information on a feed of groups, with the added (expanded) information on the Member who created each Group.

curl -i -H 'Accept: application/xml' -H 'Content-Type: application/xml'
     -H "Authorization: Bearer <!--[An OAuth2.0 auth_token has been removed from here.]-->"
     "https://<jam#>.sapjam.com/api/v1/OData/Groups?$expand=Creator"

The output from such a request can be extremely large: with all of the navigation properties included (prior to removing them for this example), the example was nearly 200 lines long for just one "group-with-creator" entry.
