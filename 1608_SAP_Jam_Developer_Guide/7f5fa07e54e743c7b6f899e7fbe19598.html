Using the OData API Reference

The SAP Jam OData API Reference provides full documentation for each endpoint, or API call. It also provides an excellent Try feature that allows API Reference users to try out each endpoint and view the results. This page provides a quick description of the information presented in the SAP Jam OData API Reference's sections. It also provides a procedure that explains how to use the Try feature.

Learning Objectives

The learning objectives for this page are:

  • Be aware of the four sections of the SAP Jam OData API Reference user interface, and understand what information is displayed in each section, including:
    • Know what information and controls are available in the banner section of the API Reference.
    • Know what information and controls are available in the navigation sidebar section of the API Reference.
    • Know what information and controls are available in the content column of the API Reference.
    • Know what information is available in the details column of the API Reference.
  • Know how to use the OData API Reference's "Try" feature.

Information in the SAP Jam OData API Reference sections

The SAP Jam OData API Reference is organized into the four sections shown in the following diagram.



Figure 123: API Reference sections

The sections of the SAP Jam OData API Reference are:

  1. The banner section contains of the title of the API Reference and the OAuth2 Token entry field, which allows you to enter the OAuth 2.0 Token generated in the SAP Jam Collaboration Administrator Guide's OAuth Client section. See the SAP Jam API Authorization page for instructions on how to generate that token.
  2. The navigation sidebar shows a menu of all of the entity sections and their endpoints. Click on any entity section title to expand that section and show its information in the content column. Click on any endpoint signature to show its documentation in the content column.
  3. The content column displays the documentation on each entity grouping and on each endpoint within that grouping. See the following section in this page, Information in the Content column for details on what information this column contains.
  4. The details column provides entity data schema details for the POST and PATCH requests and for the GET responses. See the Information in the Details column section in this page for details on what information this column contains.

Information in the Content column

The Content column provides the following information:

  • At the top of the column is a section on the $metadata file. This file is primarily used to provide information to client applications on the API entities, their properties, their navigations, and on the available service operations. This is provided primarily to aid in the import of data to the client applications. As an XML file, it is human-readable, so you can refer to the $metadata file for much of the information about the API. For information on reading the $metadata file, please see Understanding OData metadata.
  • Following the [GET] /$metadata section are a series of sections on each entity groupings covered in the SAP Jam OData API Reference. These entity groupings match the top-level titles shown in the navigation sidebar. These are "entity grouping" in that they frequently cover the API endpoints for more than one entity. For example, the Group entity set covers endpoints that relate to the Group, GroupTemplates, SystemGroupTemplates, GroupMemberships, GroupExternalObjects, GroupGadgets, and GroupGadgetObjects entities. Each of these sections begins with a quick description of the entity grouping, a diagram of the navigations available to those entities, and descriptions of each of the entities within that entity grouping.
  • Following the documentation on each entity grouping section is documentation on each endpoint of the entity grouping. That information varies depending on whether it is a POST, GET, PATCH, or DELETE operation.

    All endpoint documentation includes:

    • The signature of the endpoint, which is the HTTP verb plus the endpoint's resource path; for example: [GET] /Groups.
    • A short description of the purpose or function of the API call.
    • Descriptions of the HTTP Input Headers required to make the API call (this does not include coverage of the Accept header, as this is shown as a URL parameter for conciseness).
    • Descriptions and text boxes for inputting the various Parameters that are either required or are available for optional use with the API call.
    • A Try button that will show the results of the API call when clicked, as long as the OAuth2 Token and the required Parameters are all set.
    • A Response Type drop-down menu that allows you to set whether the API call responds in XML or JSON format.
    • The Response Messages that can be returned in response to this API call. These messages include the HTTP success code and the various HTTP error codes that may be returned for each endpoint.

Information in the Details pane

The Details pane provides the following information depending on the endpoint:

  • The Request Schema is the list of properties and their descriptions that are available for use in POST or PATCH requests.
  • The Response Schema is the list of properties and their descriptions that are returned in response to GET requests.

Note that there is also a response returned to successful POST requests. These response schemas are not shown in the interest of conciseness, but there are links to the GET endpoint documentation so the documentation of that return can be easily accessed.

How to use the OData API Reference's "Try" feature

Probably the greatest feature of our SAP Jam OData API Reference is the "Try" feature, which allows you to try out an API call and see the result immediately. To use this feature, do the following:

  1. As described in the next page, Create a group with all options set, set an OAuth client configuration for the SAP Jam API, and copy the OAuth2 Access Token.
  2. In the SAP Jam OData API Reference, paste the OAuth2 Access Token that you copied in the previous step into the OAuth2 Token field in the upper-left corner of the SAP Jam OData API Reference page.
  3. In the "content column", in the Parameters section for the API endpoint that you want to try out, fill in the available fields.
    Note that:
    • You must fill any Id fields with valid Ids, such as those retrieved by using collection-valued GET calls for that entity type.
    • You must fill any text box labeled with the API endpoint's entity type (Group, Event, etc.) with the required request payload for that endpoint.
    • You may use any allowed service query options by correctly filling in their text boxes. (See OData Query Parameters for details.) If you do not set these options, they will not be used.
    • You may select the response format that you want returned (JSON or XML). If you do not select this option, the API Reference default of the JSON format will be set.
  4. Click Try at the bottom of these fields.

    The API call will run immediately, and a pop-up window will display the response.
